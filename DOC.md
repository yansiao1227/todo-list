# TODO List 项目说明文档

## 1. 技术选型

- **编程语言**：JavaScript，理由：前端开发的主流语言，生态成熟，开发速度快。
- **框架/库**：Vue 3 + Vite，理由：组件化开发模式，Composition API 提供了更好的代码组织能力，Vite 提供了快速的开发体验和构建性能。
- **数据库/存储**：LocalStorage，理由：轻量级，无需服务器，适合本地应用的数据持久化，使用简单且足够满足 Todo 应用的需求。
- **替代方案对比**：
  - 前端框架：我认为就开发 todolist 这个简单的项目 Vue3 和 React 都可以，但我个人对 Vue3 更熟悉，开发起来效率更高。
  - 构建工具：Vite 是 Vue 3 官方推荐的构建工具，提供了快速的开发体验和构建性能。对比 Webpack，Vite 配置简单，无需繁琐的配置文件，同时支持热模块替换（HMR），开发效率高。
  - 存储替代：IndexedDB 虽然支持更复杂的数据结构，但对于简单的 Todo 应用来说，LocalStorage 已经足够且实现更简单

## 2. 项目结构设计

- **整体架构说明**：前端单页应用，采用组件化架构，通过自定义 hooks 管理状态和业务逻辑。
- **目录结构**：
  ```
  src/
    components/    # 组件目录
    hooks/         # 自定义hooks目录
    assets/        # 静态资源
    App.vue        # 应用主组件
    main.js        # 入口文件
  ```
- **模块职责说明**：
  - `components/`：包含 TodoForm、TodoList、TodoItem、TodoDetails 等 UI 组件
  - `hooks/`：包含 useTodoManager.js（任务管理逻辑）和 useNotification.js（通知提醒逻辑）
  - 采用 hooks 方式将业务逻辑从组件中抽离，提高代码复用性和可维护性

## 3. 需求细节与决策

- **描述是否必填**：否，任务描述为可选字段，当用户未输入描述时显示"无描述"。
- **已完成任务显示**：
  - 在 UI 中已完成任务的标题会显示删除线并变为灰色
  - 在任务状态标签中显示"已完成"状态
  - 任务标记为已完成后，会自动设置 notified=true 以避免触发提醒
- **任务排序逻辑**：
  - 默认按创建时间排序（最新创建的任务优先）
  - 用户可选择按优先级、截止日期或标题排序
  - 优先级排序顺序：高 > 中 > 低
  - 截止日期排序：有截止日期的任务优先，按日期先后排序
- **扩展功能设计思路**：
  - 提醒功能：使用 Web Notifications API 实现，支持设置提前 5 分钟、15 分钟、30 分钟、1 小时、1 天提醒
  - 任务过期判断：对未完成且超过截止时间的任务标记为"已过期"，并在 UI 中显示红色提醒
- **其他**：
  - 响应式设计：使用媒体查询，确保在不同设备上都能提供良好的用户体验

## 4. AI 使用说明

- **是否使用 AI 工具**：是，使用了 Trae 中的 Agent 辅助开发
- **使用 AI 的环节**：
  - 代码片段生成：生成组件模板和基础功能实现
- **AI 输出修改**：
  - 对 AI 生成的代码进行修改使其符合预期，主要是一些逻辑漏洞和 css 上的问题
  - 对整体的架构进行优化，例如在项目一开始的搭建，AI 没有考虑到功能扩展的问题，将所有的功能代码写在了 App.vue 中，后续我通过提示词引导 AI 将功能代码分离到不同的组件中，同时利用 hooks 将业务逻辑抽离出来，提高了代码的可维护性和可扩展性

## 5. 运行与测试方式

- **本地运行方式**：
  1. 安装依赖：`npm install` 或 `yarn install`
  2. 开发环境运行：`npm run dev` 或 `yarn dev`
  3. 构建生产版本：`npm run build` 或 `yarn build`
  4. 预览生产版本：`npm run preview` 或 `yarn preview`
- **已测试环境**：
  - Node.js v16+
  - 浏览器：Chrome、Firefox、Edge
- **已知问题与不足**：
  - 没有实现任务编辑功能
  - 通知权限需要用户手动授予，否则提醒功能无效
